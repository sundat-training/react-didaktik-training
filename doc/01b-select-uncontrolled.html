Aufgabe 01b

Uncontrolled Select mit useRef

🎯 Lernziel

Du löst dieselbe Aufgabe wie in 01a, aber mit useRef statt useState für den DOM-Zugriff – dem uncontrolled-Ansatz in React. Du siehst den Unterschied zwischen controlled und uncontrolled und lernst, wann welcher Ansatz sinnvoll ist.

🔧 Konzepte

<select> <option> useRef onChange map() callback props

📋 Aufgabe

  1. 1. Erstelle eine neue React-Komponente CategorySelector.tsx.
  2. 2. Definiere ein festes Array von Kategorien als Konstante.
  3. 3. Nutze useRef<HTMLSelectElement> für den Zugriff auf das DOM-Element und defaultValue statt value.
  4. 4. Zeige die gewählte Kategorie unterhalb des Dropdowns an (nutze einen kleinen State nur für die Anzeige).
  5. 5. Gib die Auswahl per Callback-Prop nach oben an die Parent-Komponente weiter.

📋 Beispieldaten zum Kopieren

const categories = ["Ordinary", "Compressed", "Moon", "Compressed Moon"];

📌 Der Unterschied zu 01a

In 01a hast du useState + value + onChange verwendet – der controlled Ansatz. Hier liest du den Wert direkt aus dem DOM via useRef und nutzt defaultValue – der uncontrolled Ansatz. Der State für die Anzeige ist nur ein "Display-State", nicht der Quell der Wahrheit für den Input-Wert.

💡 Hinweise – Code-Schnipsel

Die wichtigsten Bausteine für diese Aufgabe:

// Uncontrolled Component mit useRef:
interface CategorySelectorProps {
  title?: string;
  onCategoryChange?: (category: string) => void;
}

const selectRef = useRef<HTMLSelectElement>(null);

<select ref={selectRef} onChange={handleChange} defaultValue="">
    <option value="" disabled>Bitte wählen...</option>
    {categories.map(cat => (
        <option key={cat} value={cat}>{cat}</option>
    ))}
</select>

💡 useRef gibt ein Objekt mit .current zurück – darin liegt nach dem Rendern das DOM-Element.

💡 defaultValue setzt den initialen Wert – anders als value wird es nicht von React gesteuert.

🏁 Vollständige Lösung 
// CategorySelector.tsx

const categories = ["Ordinary", "Compressed", "Moon", "Compressed Moon"];

interface CategorySelectorProps {
  title?: string;
  onCategoryChange?: (category: string) => void;
}

function CategorySelector({ title = "Kategorie wählen", onCategoryChange }: CategorySelectorProps) {
  const selectRef = useRef<HTMLSelectElement>(null);
  const [displayValue, setDisplayValue] = useState("");

  const handleChange = () => {
    const value = selectRef.current?.value || "";
    setDisplayValue(value);
    if (value && onCategoryChange) {
      onCategoryChange(value);
    }
  };

  return (
    <div>
      <label>{title}</label>
      <select ref={selectRef} onChange={handleChange} defaultValue="">
        <option value="" disabled>Bitte wählen...</option>
        {categories.map(cat => (
          <option key={cat} value={cat}>{cat}</option>
        ))}
      </select>
      {displayValue && (
        <p>Gewählt: <strong>{displayValue}</strong></p>
      )}
    </div>
  );
}
/* CategorySelector.module.css (CSS Modules) */
.select:focus {

}

Verwendung in einer Parent-Seite:

<CategorySelector
    title="Erz-Kategorie"
    onCategoryChange={(cat) => console.log(cat)}
/>

💡 Tipps

  • Controlled = React steuert den Wert via value + onChange. Uncontrolled = das DOM steuert den Wert, du liest ihn bei Bedarf via ref aus.
  • useRef behält seine Referenz über Re-Renders hinweg – perfekt für DOM-Zugriffe ohne State.
  • Der Display-State displayValue ist nötig, weil selectRef.current.value allein kein Re-Render auslöst.
  • Faule Auswertung: Der Ref wird erst beim ersten onChange aktuell – vorher ist current null.
  • Vergleiche die Code-Menge mit Aufgabe 01a – der uncontrolled-Ansatz spart dir den value-Handler, braucht aber einen extra State für die Anzeige.

✅ Akzeptanzkriterien

  • Dropdown zeigt alle Kategorien als Optionen an
  • Platzhalter "Bitte wählen..." ist vorausgewählt (per defaultValue)
  • Die gewählte Kategorie wird unterhalb des Dropdowns angezeigt
  • Die Auswahl wird per Callback-Prop an die Parent-Komponente gemeldet
  • Es wird useRef und defaultValue verwendet – kein useState für den Select-Wert
  • Ein separater State (displayValue) dient nur der Anzeige
