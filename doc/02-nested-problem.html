02

Gekaskadete Selects: Zwei abhängige Dropdowns

🎯 Ziel dieser Einführung

Du baust zwei <select>-Dropdowns, die voneinander abhängen: Die Auswahl im ersten bestimmt, welche Optionen das zweite anzeigt. Probiere unten das fertige Verhalten aus.

🔍 Das Beispiel zum Ausprobieren

Wähle zuerst eine Kategorie, dann ein Item aus dieser Kategorie.

📖 Was passiert hier?

Reines HTML + JavaScript mit einer Datenstruktur:

// Datenquelle: Jeder Kategorie sind Items zugeordnet
var data = {
    "Ordinary": ["Veldspar", "Scordite", "Pyroxeres", "Plagioclase"],
    "Compressed": ["Compressed Veldspar", "Compressed Scordite"],
    "Moon": ["Bitumens", "Carnotite", "Cobaltite"],
    "Compressed Moon": ["Compressed Bitumens", "Compressed Carnotite"],
};

Problem: In React manipulieren wir kein DOM direkt. Stattdessen:

  • Dropdown 1 meldet die Auswahl per callback prop an die Parent-Komponente
  • Die Parent-Komponente filtert die Item-Liste und übergibt sie per props an Dropdown 2
  • React rendert Dropdown 2 automatisch neu
🧩 Übertragung nach React

Aus zwei HTML-Selects + JS-Event wird in React:

  • Zwei Child-Komponenten: CategorySelect.tsx und ItemSelect.tsx
  • Eine Parent-Komponente als "Controller", die den State hält und per props weitergibt
  • Datenfluss: Kategorie-Auswahl → onChange → Parent filtert → props → Item-Select

💡 Bevor du loslegst

  • Spiele mit der Demo – das ist das Soll-Verhalten.
  • Überlege: Wie kommst du von "zwei HTML-Selects + JS" zu "zwei React-Komponenten + Parent"?
  • Dann geht's weiter mit Aufgabe 02a (controlled) oder Aufgabe 02b (uncontrolled).
