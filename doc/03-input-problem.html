03

Input mit Validierung: Numerische Eingabe mit Fehleranzeige

🎯 Ziel dieser Einführung

Du baust ein Eingabefeld, das nur gültige Zahlen akzeptiert. Bei ungültiger Eingabe erscheint eine Fehlermeldung – mit manuellem State und Conditional Rendering.

🔍 Das Beispiel zum Ausprobieren

Gib eine Zahl ein (mindestens 1). Bei leerem Feld, Buchstaben oder Werten ≤ 0 erscheint ein Fehler.

Warte auf Eingabe...
📖 Was passiert hier?

Reines HTML + JavaScript:

// Prüft bei jeder Eingabe
input.addEventListener('input', function() {
    var raw = this.value.trim();
    var parsed = parseInt(raw, 10);

    if (raw === '' || isNaN(parsed) || parsed <= 0) {
        // Fehler anzeigen
        errorEl.textContent = 'Bitte gib eine positive ganze Zahl ein.';
        inputEl.classList.add('error');
        outputEl.textContent = '';
    } else {
        // Gültig: Fehler aus
        errorEl.textContent = '';
        inputEl.classList.remove('error');
        outputEl.textContent = 'Gültig: ' + parsed;
    }
});

Problem: In React gibt es kein classList.add und kein direktes textContent. Du musst:

  • Den Wert mit useState verwalten
  • Einen Error-String als State-Variable führen
  • Den Error-String per && oder Ternary conditional rendern
  • Eine CSS-Klasse per Klammertoggeln setzen
🧩 Übertragung nach React

Aus dem JS-Code wird in React:

  • input.value → value={value} onChange={...} an einen string-State
  • parseInt → parseInt(raw, 10) im Event-Handler
  • CSS-Klasse togglen → className={isValid ? "" : "error"}
  • Fehlertext setzen → setErrorMessage("...") (React rendert automatisch)
  • Callback feuern → nur bei gültigem Wert

💡 Bevor du loslegst

  • Probiere die Demo: was passiert bei Buchstaben? Bei negativen Zahlen? Bei 0?
  • Überlege: Wie bildest du "CSS-Klasse togglen" in React ab?
