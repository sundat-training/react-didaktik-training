Aufgabe 03b

Input mit Validierung & controlled

🎯 Lernziel

Du löst dieselbe Aufgabe wie in 03, aber mit einer klaren Trennung: Der Parent hält den Wert-State, die Child-Komponente ist ein reiner controlled Input mit Callback-Prop. Du siehst, wie der controlled-Ansatz die Datenhoheit beim Parent belässt.

🔧 Konzepte

interface Props useState onChange Controlled Component parseInt Conditional Rendering Callback-Props

📋 Aufgabe

  1. 1. Erstelle QuantityInput.tsx mit <input type="number"> als controlled Component.
  2. 2. Der Parent übergibt value und onChange – die Komponente hat keinen eigenen State für den Wert.
  3. 3. Validiere im onChange-Handler: parse mit parseInt(), setze Error-String bei Ungültigkeit.
  4. 4. Zeige den Error-String per Conditional Rendering.
  5. 5. Feuere die onChange-Callback nur bei gültigem Wert.
  6. 6. Füge disabled?: boolean und minValue?: number hinzu.

📌 Der Unterschied zu 03

In 03 verwaltet die Input-Komponente ihren eigenen Wert-State. Hier wird der Wert vom Parent gesteuert (controlled). Die Komponente hat keinen useState mehr für inputValue – der Wert kommt von außen. Das macht die Komponente wiederverwendbarer, da der Parent die Datenhoheit behält.

💡 Hinweise – Code-Schnipsel

Die wichtigsten Bausteine:

// Controlled Component: value + onChange von außen
interface QuantityInputProps {
  value: string;
  onChange?: (value: number) => void;
  disabled?: boolean;
  minValue?: number;
}

<input type="number"
       className={...}
       onChange={handleChange}
       value={value}
       disabled={disabled} />

// Validierung + Callback im Handler:
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  const raw = e.target.value;
  const num = parseInt(raw, 10);

  if (raw === "" || isNaN(num) || num < minValue) {
    setErrorMessage("...");
    return;
  }

  setErrorMessage(null);
  onChange?.(num);
};

💡 Der Parent ruft die Komponente so auf:
<QuantityInput value={qty} onChange={setQty} />

🏁 Vollständige Lösung 
{/* QuantityInput.tsx */}

interface QuantityInputProps {
  value: string;
  onChange?: (value: number) => void;
  disabled?: boolean;
  minValue?: number;
  label?: string;
}

function QuantityInput({ value, onChange, disabled, minValue = 1, label = "Menge" }: QuantityInputProps) {
  const [errorMessage, setErrorMessage] = useState<string | null>(null);

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const raw = e.target.value;

    if (raw.trim() === "") {
      setErrorMessage("Feld darf nicht leer sein");
      return;
    }

    const num = parseInt(raw, 10);
    if (isNaN(num)) {
      setErrorMessage("Bitte eine gültige ganze Zahl eingeben");
      return;
    }

    if (num < minValue) {
      setErrorMessage(`Wert muss >= ${minValue} sein`);
      return;
    }

    setErrorMessage(null);
    onChange?.(num);
  };

  return (
    <div className="mb-3">
      <label className="form-label fw-bold">{label}</label>
      <input type="number"
             className={`form-control ${errorMessage ? "is-invalid" : ""}`}
             onChange={handleChange}
             value={value}
             disabled={disabled}
             min={minValue} />
      {errorMessage && (
        <div className="text-danger small mt-1">{errorMessage}</div>
      )}
    </div>
  );
}
{/* QuantityInput.module.css */}
.is-invalid {
    border-color: #dc3545;
}
.is-invalid:focus {
    box-shadow: 0 0 0 2px rgba(220, 53, 69, 0.25);
}

Parent-Seite (Beispiel):

function OrderPage() {
  const [qty, setQty] = useState("");

  return (
    <div>
      <QuantityInput
        value={qty}
        onChange={(num) => setQty(num.toString())}
      />
      {qty && <p>Bestellte Menge: {qty}</p>}
    </div>
  );
}

💡 Tipps

  • Im Gegensatz zu 03 gibt es kein useState für den Wert in der Komponente – der Wert kommt vom Parent.
  • Der Error-String hingegen bleibt lokal in der Komponente – das ist sinnvoll, da der Parent nicht wissen muss, ob aktuell ein Fehler angezeigt wird.
  • Leeres Feld = Fehler. Entweder du blockierst es oder erlaubst es als "noch nichts eingegeben".

✅ Akzeptanzkriterien

  • Akzeptiert nur positive ganze Zahlen > 0
  • Zeigt eine rote Fehlermeldung bei leerem Feld, 0 oder negativen Werten
  • Blockiert die Weitergabe des Wertes bei ungültiger Eingabe
  • Das Feld kann von außen deaktiviert werden
  • Der Wert wird vom Parent gesteuert (controlled component)
