Vorbereitung

Grundprojekt aufsetzen

🎯 Ziel

Du richtest ein neues Vite + React + TypeScript Projekt mit Router ein. Dieses Projekt verwendest du für alle folgenden Aufgaben. Ein sauberes Setup ist die halbe Miete – danach kannst du dich ganz auf die Komponenten konzentrieren.

🔧 Konzepte

Vite React TypeScript react-router-dom Tailwind CSS tsconfig paths

📋 Aufgabe

  1. 1. Erstelle ein neues Vite-Projekt mit React + TypeScript Template.
  2. 2. Installiere React Router DOM und Tailwind CSS.
  3. 3. Lege die empfohlene Projektstruktur an.
  4. 4. Richte den Router mit einer Startseite und einer About-Seite ein.
  5. 5. Erstelle ein Root-Layout mit Navigation (Navbar + Outlet).
  6. 6. Optional: Richte TypeScript-Pfad-Aliase für saubere Imports ein.

📋 Terminal-Befehle zum Kopieren

# 1. Projekt erstellen
npm create vite@latest mein-projekt -- --template react-ts
cd mein-projekt
npm install

# 2. React Router + Tailwind hinzufügen
npm install react-router-dom
npm install -D tailwindcss @tailwindcss/vite

# 3. (Optional) Pfad-Alias-Unterstützung
npm install -D vite-tsconfig-paths
💡 Empfohlene Projektstruktur

So sollte deine src/-Struktur aussehen:

src/
├── main.tsx
├── App.tsx
├── router/
│   └── router.tsx
├── pages/
│   ├── Home.tsx
│   └── About.tsx
├── components/
│   └── layout/
│       └── Navbar.tsx
└── assets/

Diese Struktur ist bewusst schlank gehalten – für jede Aufgabe kommt später ein eigener Ordner unter pages/ dazu. Die components/-Ordner sammeln wiederverwendbare UI-Elemente.

🔧 Router und Layout einrichten

Erstelle src/router/router.tsx:

import { createBrowserRouter } from 'react-router-dom';
import RootLayout from '../App';
import Home from '../pages/Home';
import About from '../pages/About';

export const router = createBrowserRouter([
  {
    path: '/',
    element: <RootLayout />,
    children: [
      { index: true, element: <Home /> },
      { path: 'about', element: <About /> },
    ],
  },
]);

Tailwind in vite.config.ts aktivieren:

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [react(), tailwindcss()],
});

Tailwind-Direktive in src/index.css (Datei anlegen):

@import "tailwindcss";

src/main.tsx:

import React from 'react';
import ReactDOM from 'react-dom/client';
import { RouterProvider } from 'react-router-dom';
import { router } from './router/router';

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <RouterProvider router={router} />
  </React.StrictMode>
);

src/App.tsx – Root-Layout mit Navbar:

import Navbar from './components/layout/Navbar';
import { Outlet } from 'react-router-dom';

function RootLayout() {
  return (
    <div className="min-h-screen bg-zinc-50">
      <Navbar />
      <main>
        <Outlet />
      </main>
    </div>
  );
}

export default RootLayout;
🏁 Vollständiges Setup

Navbar-Komponente (src/components/layout/Navbar.tsx):

import { NavLink } from 'react-router-dom';

export default function Navbar() {
  return (
    <nav className="border-b">
      <div className="max-w-7xl mx-auto px-6 py-4
                  flex justify-between items-center">
        <h1 className="text-2xl font-bold">
          React Training
        </h1>

            to="/"

  );
}

Bonus: TypeScript-Pfad-Aliase

vite.config.ts:

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tsconfigPaths from 'vite-tsconfig-paths';

export default defineConfig({
  plugins: [react(), tsconfigPaths()],
});

tsconfig.json (Abschnitt compilerOptions ergänzen):

{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}

Danach sind Imports wie import Navbar from '@/components/layout/Navbar' möglich – keine relativen Kaskaden mehr.

💡 Tipps

  • Vite startet mit npm run dev – der Dev-Server hat Hot Module Replacement (HMR). Änderungen siehst du sofort im Browser.
  • React Router v7 (react-router-dom) verwendet createBrowserRouter – das ist der empfohlene Weg für TypeScript-Projekte.
  • Der <Outlet /> im Layout rendert die jeweilige Kind-Route. Ohne ihn wird nichts angezeigt.
  • NavLink von React Router bietet automatische isActive-Erkennung – ideal für Navigationsmenüs.
  • Tailwind wird als Vite-Plugin eingebunden – die CSS-Direktive @import "tailwindcss" in src/index.css aktiviert alle Utility-Klassen global.
  • Mit Tailwind entfällt das Schreiben von eigenem CSS für Layout und Abstände – alles über Klassen wie flex, gap-4, p-6.

✅ Akzeptanzkriterien

  • npm run dev startet ohne Fehler
  • Tailwind CSS ist installiert und aktiviert (Utility-Klassen wie flex, text-blue-600 werden erkannt)
  • Die Startseite zeigt "React Component Training" an
  • Die About-Seite ist über die Navigation erreichbar
  • Die Navbar hebt die aktive Seite farblich hervor
  • Die Ordnerstruktur folgt dem empfohlenen Schema
  • Bonus: @/-Imports funktionieren (wenn Alias eingerichtet)
