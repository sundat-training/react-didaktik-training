Component State: Alles zusammen – eine Seite, mehrere Komponenten

🎯 Ziel dieser Einführung

Du kombinierst Selects, Inputs und Tabellen zu einer funktionsfähigen Seite. Eine zentrale "Controller"-Komponente orchestriert den State und verteilt Daten per Props an Kinder.

🔍 Das Beispiel zum Ausprobieren

Wähle ein Item, gib Menge und Gebühr ein – der Gesamtwert wird berechnet. Alle Felder hängen zusammen und melden Änderungen an die Zentrale.

Warte auf Eingabe...
📖 Was passiert hier?

Mehrere Eingabefelder → ein Ergebnis:

// Jede Änderung triggert die Berechnung
function calculate() {
    var price = parseFloat(selectedItem.dataset.price);
    var qty = parseInt(qtyInput.value);
    var fee = parseFloat(feeInput.value);
    var total = price * qty * (1 + fee / 100);
    showResult(total);
}

selectedItem.addEventListener('change', calculate);
qtyInput.addEventListener('input', calculate);
feeInput.addEventListener('input', calculate);

In React: Eine Parent-Komponente hält selectedItem, quantity, fee. Jede Child-Komponente meldet Änderungen per Callback-Prop – die Parent aktualisiert den State und React rendert neu.

🧩 Übertragung nach React
  • Controller-Komponente: Calculator.tsx hält den gesamten State mit useState
  • Kinder: CategorySelect, QuantityInput, DualInput, OrderTable – alle aus vorherigen Aufgaben
  • Datenfluss: Child → Callback-Prop → Parent-State → Props → Child
  • Formel: result = itemPrice * quantity * (1 + fee / 100)
