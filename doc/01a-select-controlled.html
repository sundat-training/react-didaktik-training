Aufgabe 01a

Controlled Select

🎯 Lernziel

Du erstellst eine Komponente mit einem <select>-Dropdown und steuerst es über useState + onChange. Jede Auswahl wird per Callback-Prop an die Parent-Komponente gemeldet.

🔧 Konzepte

<select> <option> onChange ChangeEvent map() callback props useState

📋 Aufgabe

  1. 1. Erstelle eine neue React-Komponente CategorySelector.tsx.
  2. 2. Definiere ein festes Array von Kategorien als Konstante (z.B. "Ordinary", "Compressed", "Moon", "Compressed Moon").
  3. 3. Nutze useState für die aktuell gewählte Kategorie und baue ein <select> mit value={selectedCategory} und onChange={handleChange}.
  4. 4. Zeige die gewählte Kategorie unterhalb des Dropdowns an.
  5. 5. Gib die Auswahl per Callback-Prop (z.B. onCategoryChange) nach oben an die Parent-Komponente weiter.

📋 Beispieldaten zum Kopieren

const categories = ["Ordinary", "Compressed", "Moon", "Compressed Moon"];
💡 Hinweise – Code-Schnipsel

Die wichtigsten Bausteine für diese Aufgabe:

// Controlled Component mit onChange + useState:
interface CategorySelectorProps {
  title?: string;
  onCategoryChange?: (category: string) => void;
}

<select value={selectedCategory} onChange={handleChange}>
    <option value="" disabled>Bitte wählen...</option>
    {categories.map(cat => (
        <option key={cat} value={cat}>{cat}</option>
    ))}
</select>

💡 useState gibt dir ein Tupel [wert, setter] – der Setter triggert ein Re-Render.

💡 React.ChangeEvent<HTMLSelectElement> liefert den neuen Wert über e.target.value.

🏁 Vollständige Lösung 
// CategorySelector.tsx

const categories = ["Ordinary", "Compressed", "Moon", "Compressed Moon"];

interface CategorySelectorProps {
  title?: string;
  onCategoryChange?: (category: string) => void;
}

function CategorySelector({ title = "Kategorie wählen", onCategoryChange }: CategorySelectorProps) {
  const [selectedCategory, setSelectedCategory] = useState("");

  const handleChange = (e: React.ChangeEvent<HTMLSelectElement>) => {
    const value = e.target.value;
    setSelectedCategory(value);
    if (value && onCategoryChange) {
      onCategoryChange(value);
    }
  };

  return (
    <div>
      <label>{title}</label>
      <select value={selectedCategory} onChange={handleChange}>
        <option value="" disabled>Bitte wählen...</option>
        {categories.map(cat => (
          <option key={cat} value={cat}>{cat}</option>
        ))}
      </select>
      {selectedCategory && (
        <p>Gewählt: <strong>{selectedCategory}</strong></p>
      )}
    </div>
  );
}
/* CategorySelector.module.css (CSS Modules) */
.select:focus {
    border-color: #4361ee;
    box-shadow: 0 0 0 2px rgba(67, 97, 238, 0.2);
}

Verwendung in einer Parent-Seite:

<CategorySelector
    title="Erz-Kategorie"
    onCategoryChange={(cat) => console.log(cat)}
/>

💡 Tipps

  • Ein Platzhalter-Eintrag ("Bitte wählen...") braucht value="" und disabled.
  • Ein Controlled Component bedeutet: React steuert den Wert über value + onChange – der State ist die "Single Source of Truth".
  • useState ist ein Hook – er muss immer auf der obersten Ebene der Funktion aufgerufen werden, nicht in Schleifen oder Bedingungen.
  • Der Callback wird nur bei gültiger Auswahl gefeuert (value && onCategoryChange).
  • React rendert bei jeder State-Änderung neu – das ist erwünscht und kein Fehler.

✅ Akzeptanzkriterien

  • Dropdown zeigt alle Kategorien als Optionen an
  • Platzhalter "Bitte wählen..." ist vorausgewählt
  • Die gewählte Kategorie wird unterhalb des Dropdowns angezeigt
  • Die Auswahl wird per Callback-Prop an die Parent-Komponente gemeldet
  • Bei leerer Auswahl (Platzhalter) wird kein Event gefeuert
  • Die Komponente hat eine title-Prop für die Beschriftung
