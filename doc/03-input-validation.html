Aufgabe 03

Input mit Validierung

🎯 Lernziel

Du erstellst ein numerisches Eingabefeld mit eigener Validierung. Du lernst, wie man Eingaben parst, Fehlerzustände verwaltet und sie benutzerfreundlich anzeigt – mit useState und Conditional Rendering.

🔧 Konzepte

<input type="number"> useState onChange Conditional Rendering parseInt CSS class toggling Callback-Props

📋 Aufgabe

  1. 1. Erstelle die Komponente QuantityInput.tsx mit einem <input type="number">.
  2. 2. Bei jeder Eingabe (onChange) parst du den Wert mit parseInt(). Ist der Wert ungültig (keine Zahl, ≤ 0), setzt du einen Error-String.
  3. 3. Zeige den Error-String als rote Meldung nur an, wenn ein Fehler vorliegt (Conditional Rendering mit &&).
  4. 4. Gib den gültigen Zahlenwert per Callback-Prop nach oben. Bei ungültiger Eingabe feuere kein Event.
  5. 5. Füge eine Prop disabled?: boolean hinzu. Wenn true, wird das Input-Feld deaktiviert.
  6. 6. Bonus: Füge Props label?: string und minValue?: number hinzu, um die Komponente allgemein nutzbar zu machen.
💡 Hinweise – Code-Schnipsel

Die wichtigsten Bausteine:

// onChange = feuert bei jeder Änderung:
<input type="number"
       onChange={handleChange}
       value={inputValue}
       disabled={disabled} />

const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const raw = e.target.value;
    const num = parseInt(raw, 10);
    if (!isNaN(num) && num > 0) {
        // gültig
    } else {
        // Fehler setzen
    }

// Conditional Rendering für Fehler:
{errorMessage && (
    <div style={{ color: "red" }}>{errorMessage}</div>

🏁 Vollständige Lösung 
{/* QuantityInput.tsx */}

interface QuantityInputProps {
  label?: string;
  minValue?: number;
  disabled?: boolean;
  onQuantityChanged?: (value: number) => void;
}

function QuantityInput({ label = "Menge", minValue = 1, disabled, onQuantityChanged }: QuantityInputProps) {
  const [inputValue, setInputValue] = useState("");
  const [errorMessage, setErrorMessage] = useState<string | null>(null);

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const raw = e.target.value;
    setInputValue(raw);

    if (raw.trim() === "") {
      setErrorMessage("Feld darf nicht leer sein");
      return;
    }

    const num = parseInt(raw, 10);
    if (isNaN(num)) {
      setErrorMessage("Bitte eine gültige ganze Zahl eingeben");
      return;
    }

    if (num < minValue) {
      setErrorMessage(`Wert muss >= ${minValue} sein`);
      return;
    }

    setErrorMessage(null);
    onQuantityChanged?.(num);
  };

  return (
    <div className="mb-3">
      <label className="form-label fw-bold">{label}</label>
      <input type="number"
             className={`form-control ${errorMessage ? "is-invalid" : ""}`}
             onChange={handleChange}
             value={inputValue}
             disabled={disabled}
             min={minValue} />
      {errorMessage && (
        <div className="text-danger small mt-1">{errorMessage}</div>
      )}
    </div>
  );
}

}

}

💡 Tipps

  • onChange feuert bei jeder Änderung – perfekt für Live-Validierung.
  • e.target.value ist immer ein string. Du musst selbst parsen.
  • Ein leeres Feld ist auch ein Sonderfall: Soll es erlaubt sein oder ein Fehler?
  • Zusätzlich zur Textmeldung: Füge eine CSS-Klasse is-invalid hinzu, die das Feld rot umrandet.

✅ Akzeptanzkriterien

  • Akzeptiert nur positive ganze Zahlen > 0
  • Zeigt eine rote Fehlermeldung bei leerem Feld, 0 oder negativen Werten
  • Blockiert die Weitergabe des Wertes bei ungültiger Eingabe
  • Das Feld kann von außen deaktiviert werden
  • Das Feld hat eine rote Umrandung bei Fehler (CSS-Klasse .is-invalid)
