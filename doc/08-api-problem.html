08

API-Integration: Daten von einer REST-API laden

🎯 Ziel dieser Einführung

Du rufst Daten von einer externen REST-API ab und stellst sie in der UI dar – mit Ladezustand, Fehlerbehandlung und JSON-Parsing.

🔍 Das Beispiel zum Ausprobieren

Klicke auf "Daten laden" (simulierter API-Call). Beobachte Loading-Spinner, Datenanzeige und Fehlerfall.

Noch keine Daten geladen.
📖 Was passiert hier?

Simulierter asynchroner API-Call mit setTimeout:

// Simulierter API-Call
async function loadData(shouldFail) {
    showLoading();                     // Spinner anzeigen
    try {
        var response = await fakeApi(shouldFail); // setTimeout als Promise
        showData(response);            // Daten rendern
    } catch (err) {
        showError(err.message);        // Fehler anzeigen
    }
}

// In React wird daraus:
function TypeInfo() {
    const [status, setStatus] = useState<"loading" | "error" | "success">("idle");
    const [data, setData] = useState(null);
    const [error, setError] = useState(null);

    useEffect(() => {
        setStatus("loading");
        fetch("https://api.example.com/data")
            .then(res => res.json())
            .then(json => { setData(json); setStatus("success"); })
            .catch(err => { setError(err.message); setStatus("error"); });
    }, []);
}

Problem: In React gibt es kein DOM-Manipulation. status, data und error sind State-Variablen – React rendert automatisch die passende Ansicht.

🧩 Übertragung nach React
  • fetch(): Browser-API für HTTP-Requests – kein spezieller Service nötig
  • Laden: async/await in useEffect
  • JSON parsen: response.json()
  • Paginierung: Mehrere Requests nacheinander oder parallel mit Promise.all
  • Drei UI-Zustände: {status === "loading" && (...)} / {status === "error" && (...)} / {status === "success" && (...)}

💡 Bevor du loslegst

  • Denk an die drei Zustände: Loading, Error, Success – jeder braucht eine eigene UI.
  • Wie fängst du Fehler bei API-Calls ab, ohne dass die App abstürzt?
