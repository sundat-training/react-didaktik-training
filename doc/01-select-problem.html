01

Select: Umsetzen einer simplen Select-Komponente

🎯 Ziel dieser Aufgabe

Du baust eine React-Komponente mit einem <select>-Dropdown. Das klingt simpel – aber in React läuft Datenbindung anders als in reinem HTML/JS. Probiere unten das fertige Verhalten aus, dann setzt du es in React um.

🔍 Das Beispiel zum Ausprobieren

Wähle eine Kategorie aus dem Dropdown. Die Auswahl wird sofort angezeigt – genau dieses Verhalten soll deine React-Komponente später haben.

📖 Was passiert hier?

Reines HTML + JavaScript:

<!-- Das HTML -->
<select id="demoSelect">
    <option value="" disabled selected>Bitte wählen...</option>
    <option value="Ordinary">Ordinary</option>
    <option value="Compressed">Compressed</option>
    <option value="Moon">Moon</option>
    <option value="Compressed Moon">Compressed Moon</option>
</select>
<p id="demoOutput"></p>

<!-- Das JavaScript -->
<script>
document.getElementById('demoSelect')
    .addEventListener('change', function(e) {
        var output = document.getElementById('demoOutput');
        output.textContent = e.target.value
            ? 'Gewählt: ' + e.target.value
            : '';
    });
</script>

Problem: In React gibt es kein document.getElementById und kein direktes DOM-Manipulation. Du musst State und Events nutzen: useState + onChange. Das DOM wird von React für dich verwaltet.

🧩 Übertragung nach React

Aus dem HTML/JS wird in React:

  • HTML-Teil → bleibt fast gleich, nur id und JS-Events entfallen
  • e.target.value → wird zu useState + onChange
  • textContent setzen → wird zu einer State-Variable, die React automatisch rendert
  • EventListener → wird zu onChange (controlled) oder useRef + onChange (uncontrolled)

Das Ziel: Dieselbe Benutzererfahrung – aber ohne DOM-API, nur mit React State & Props.

💡 Bevor du loslegst

  • Spiele mit dem Demo-Dropdown oben – das ist das Soll-Verhalten.
  • Überlege: Wie würdest du "Auswahl speichern und anzeigen" ohne DOM-API umsetzen?
  • Dann geht's weiter mit Aufgabe 01a (controlled) oder Aufgabe 01b (uncontrolled).
