04

Gekoppelte Inputs: Slider + Zahlenfeld synchron

🎯 Ziel dieser Einführung

Du kombinierst einen Slider und ein Zahlenfeld, die denselben Wert steuern. Eine Änderung im Slider aktualisiert sofort das Zahlenfeld – und umgekehrt. Du lernst bidirektionale Synchronisation mit useState.

🔍 Das Beispiel zum Ausprobieren

Bewege den Slider oder tippe eine Zahl ein – beide bleiben synchron.

42,0 %
Wert: 42,0 %
📖 Was passiert hier?

Zwei Inputs, ein Wert – Synchronisation in beide Richtungen:

// Slider → Number
slider.addEventListener('input', function() {
    numberEl.value = this.value;       // Number aktualisieren
    updateDisplay(this.value);
});

// Number → Slider
numberEl.addEventListener('change', function() {
    var val = parseFloat(this.value);
    if (!isNaN(val) && val >= 0 && val <= 100) {
        sliderEl.value = val;           // Slider aktualisieren
        updateDisplay(val);
    }
});

Problem: In React manipulieren wir keine DOM-Elemente direkt. Beide Inputs binden an dieselbe State-Variable. Ändert sich die Variable (egal durch welchen Input), rendert React beide neu.

🧩 Übertragung nach React

Aus zwei DOM-Elementen + JS wird in React:

  • Eine Komponente DualInput mit zwei Input-Feldern
  • Eine State-Variable const [value, setValue] = useState(50) – beide Inputs lesen von ihr
  • Zwei Handler: handleSliderChange und handleNumberChange – beide schreiben in value
  • Ein Callback: onValueChange als Prop

💡 Bevor du loslegst

  • Probiere die Demo: Slider bewegen, dann Zahl eintippen – alles bleibt synchron.
  • Welchen Event-Typ nutzt du für den Slider (flüssig)? Welchen für das Number-Feld (präzise)?
