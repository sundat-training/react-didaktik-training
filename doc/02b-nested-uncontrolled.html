Aufgabe 02b

Gekaskadete Selects uncontrolled (useRef)

🎯 Lernziel

Du löst dieselbe Aufgabe wie in 02a, aber mit useRef statt useState in den Child-Komponenten – dem uncontrolled-Ansatz in React. Du siehst, wie useRef den Zugriff auf DOM-Werte ohne zusätzlichen State ermöglicht.

🔧 Konzepte

interface Props useRef onChange Parent→Child Datenfluss Record<K,V> Conditional Rendering key

📋 Aufgabe

  1. 1. Erstelle zwei Komponenten: CategorySelect.tsx (Kategorie-Auswahl) und ItemSelect.tsx (Item-Auswahl).
  2. 2. Beide Komponenten nutzen useRef<HTMLSelectElement> für den DOM-Zugriff.
  3. 3. CategorySelect gibt die gewählte Kategorie per Callback-Prop nach oben – ausgelesen via ref.current.value.
  4. 4. ItemSelect nimmt per Props eine Liste von Items entgegen und zeigt sie im Dropdown an.
  5. 5. Erstelle eine Parent-Seite, die beide Komponenten einbettet. Bei Kategorie-Wechsel wird die Item-Liste neu gesetzt.
  6. 6. Verwende ein Record<string, string[]> als Datenquelle.
  7. 7. Sorge dafür, dass bei Kategorie-Wechsel die Item-Auswahl zurückgesetzt wird.

📋 Beispieldaten zum Kopieren

const oreData: Record<string, string[]> = {
  "Ordinary": ["Veldspar", "Scordite", "Pyroxeres", "Plagioclase"],
  "Compressed": ["Compressed Veldspar", "Compressed Scordite", "Compressed Pyroxeres"],
  "Moon": ["Bitumens", "Carnotite", "Cobaltite"],
  "Compressed Moon": ["Compressed Bitumens", "Compressed Carnotite"],
};

📌 Der Unterschied zu 02a

In 02a nutzen beide Childs useState + onChange + value – controlled components. Hier setzen beide auf useRef + onChange und lesen den Wert direkt aus dem DOM – weniger State, gleiche Schnittstelle nach außen.

💡 Hinweise – Code-Schnipsel

Die wichtigsten Bausteine:

// CategorySelect mit useRef:
const selectRef = useRef<HTMLSelectElement>(null);

const handleChange = () => {
  const cat = selectRef.current?.value ?? "";
  if (cat && onCategoryChanged) onCategoryChanged(cat);
};

<select ref={selectRef} onChange={handleChange} defaultValue="">
  ...
</select>

// ItemSelect mit useRef + disabled:
const selectRef = useRef<HTMLSelectElement>(null);

const handleChange = () => {
  const item = selectRef.current?.value ?? "";
  if (item && onItemSelected) onItemSelected(item);
};

<select ref={selectRef} onChange={handleChange}
        defaultValue="" disabled={disabled}>
  ...
</select>

// Parent bleibt gleich:
<CategorySelect onCategoryChanged={handleCategoryChanged} />
<ItemSelect items={currentItems} onItemSelected={handleItemSelected}
            disabled={!selectedCategory}
            key={selectedCategory} />

💡 Die Callback-Schnittstelle ändert sich nicht – nur die interne Implementierung der Childs wird anders.

🏁 Vollständige Lösung 
{/* CategorySelect.tsx */}

interface CategorySelectProps {
  onCategoryChanged?: (category: string) => void;
}

function CategorySelect({ onCategoryChanged }: CategorySelectProps) {
  const selectRef = useRef<HTMLSelectElement>(null);

  const categories = ["Ordinary", "Compressed", "Moon", "Compressed Moon"];

  const handleChange = () => {
    const cat = selectRef.current?.value ?? "";
    if (cat && onCategoryChanged) onCategoryChanged(cat);
  };

  return (
    <div className="mb-3">
      <label className="form-label fw-bold">Kategorie</label>
      <select className="form-select"
              ref={selectRef}
              onChange={handleChange}
              defaultValue="">
        <option value="" disabled>Bitte wählen...</option>
        {categories.map(cat => (
          <option key={cat} value={cat}>{cat}</option>
        ))}
      </select>
    </div>
  );
}
{/* ItemSelect.tsx */}

interface ItemSelectProps {
  items: string[];
  disabled?: boolean;
  onItemSelected?: (item: string) => void;
}

function ItemSelect({ items, disabled, onItemSelected }: ItemSelectProps) {
  const selectRef = useRef<HTMLSelectElement>(null);

  const handleChange = () => {
    const item = selectRef.current?.value ?? "";
    if (item && onItemSelected) onItemSelected(item);
  };

  return (
    <div className="mb-3">
      <label className="form-label fw-bold">Item</label>
      <select className="form-select"
              ref={selectRef}
              onChange={handleChange}
              defaultValue=""
              disabled={disabled}>
        <option value="" disabled>Bitte wählen...</option>
        {items.map(item => (
          <option key={item} value={item}>{item}</option>
        ))}
      </select>
    </div>
  );
}

Parent-Seite:

{/* CascadingSelectPage.tsx */}

function CascadingSelectPage() {
  const [selectedCategory, setSelectedCategory] = useState("");
  const [selectedItem, setSelectedItem] = useState("");
  const [currentItems, setCurrentItems] = useState<string[]>([]);

  const oreData: Record<string, string[]> = {
    "Ordinary": ["Veldspar", "Scordite", "Pyroxeres", "Plagioclase"],
    "Compressed": ["Compressed Veldspar", "Compressed Scordite"],
  };

  const handleCategoryChanged = (category: string) => {
    setSelectedCategory(category);
    setCurrentItems(oreData[category] ?? []);
    setSelectedItem("");
  };

  const handleItemSelected = (item: string) => {
    setSelectedItem(item);
    console.log(`Item: ${item}`);
  };

  return (
    <div>
      <CategorySelect onCategoryChanged={handleCategoryChanged} />
      <ItemSelect items={currentItems}
                  disabled={!selectedCategory}
                  onItemSelected={handleItemSelected}
                  key={selectedCategory} />
      {selectedItem && (
        <p>Auswahl: {selectedCategory} → {selectedItem}</p>
      )}
    </div>
  );
}

💡 Tipps

  • Die Parent-Komponente bleibt fast identisch zu 02a – nur die Childs werden mit useRef anders implementiert.
  • Bei useRef liest du den Wert erst bei Bedarf aus – es gibt kein value-Binding auf dem <select>.
  • key auf ItemSelect sorgt dafür, dass React die Komponente bei Kategorie-Wechsel neu aufbaut und der interne Ref-Zustand zurückgesetzt wird.
  • Vergleiche die Code-Menge von CategorySelect in 02a vs. 02b.

✅ Akzeptanzkriterien

  • Kategorie-Dropdown zeigt alle Kategorien an
  • Item-Dropdown zeigt nur Items der gewählten Kategorie
  • Bei Kategorie-Wechsel wird die Item-Auswahl zurückgesetzt
  • Item-Dropdown ist deaktiviert, solange keine Kategorie gewählt ist
  • Die Auswahl beider Dropdowns wird auf der Seite angezeigt
  • Beide Childs verwenden useRef – kein useState für den Wert
