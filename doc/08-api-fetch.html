Aufgabe 08

API-Integration mit fetch

🎯 Lernziel

Du rufst Daten von einer externen REST-API ab und stellst sie in der UI dar. Du lernst die Verwendung von fetch asynchronen API-Calls (async/await), die Verarbeitung von JSON-Responses und die drei UI-Zustände Loading, Error, Success.

🔧 Konzepte

useState useEffect fetch() async/await Conditional Rendering try/catch

📋 Aufgabe

  1. 1. Erstelle eine Komponente TypeInfo.tsx, die eine typeId: number als Prop erhält.
  2. 2. Nutze useState für drei Zustände: status: "loading" | "error" | "success", data, error.
  3. 3. Rufe in useEffect die EVE Online ESI API auf: https://esi.evetech.net/latest/universe/types/{typeId}/.
  4. 4. Zeige einen Spinner während des Ladens, die Daten bei Erfolg und eine Fehlermeldung bei Fehler.
  5. 5. Fange HTTP-Fehler (z.B. 404) und Netzwerkfehler mit try/catch ab.
  6. 6. Bonus: Verwende AbortController, um den Request abzubrechen, wenn die Komponente unmounted wird.

📋 Beispieldaten zum Kopieren

// TypeScript-Datenmodell
interface TypeInfo {
    type_id: number;
    name: string;
    description: string;
}

type Status = "loading" | "error" | "success";

// fetch-Aufruf (ESI API)
const response = await fetch(`https://esi.evetech.net/latest/universe/types/${typeId}/`);
if (!response.ok) throw new Error(`HTTP ${response.status}`);
const data: TypeInfo = await response.json();

// TypeInfo-Komponente (Grundstruktur)
function TypeInfo({ typeId }: { typeId: number }) {
    const [status, setStatus] = useState<Status>("loading");
    const [data, setData] = useState<TypeInfo | null>(null);
    const [error, setError] = useState<string | null>(null);

    useEffect(() => {
        setStatus("loading");
        fetch(`https://esi.evetech.net/latest/universe/types/${typeId}/`)
            .then(res => {
                if (!res.ok) throw new Error(`HTTP ${res.status}`);
                return res.json();
            })
            .then(json => {
                setData(json);
                setStatus("success");
            })
            .catch(err => {
                setError(err.message);
                setStatus("error");
            });
    }, [typeId]);

    if (status === "loading") return <p>Lade Daten...</p>;
    if (status === "error") return <p style={{color:"red"}}>Fehler: {error}</p>;
    return <div><h2>{data?.name}</h2><p>{data?.description}</p></div>;
}

🏁 Vollständige Lösung 
// TypeInfo.tsx
import { useState, useEffect } from "react";

interface TypeInfo {
    type_id: number;
    name: string;
    description: string;
}

type Status = "loading" | "error" | "success";

export function TypeInfo({ typeId }: { typeId: number }) {
    const [status, setStatus] = useState<Status>("loading");
    const [data, setData] = useState<TypeInfo | null>(null);
    const [error, setError] = useState<string | null>(null);

    useEffect(() => {
        setStatus("loading");
        const controller = new AbortController();

        fetch(`https://esi.evetech.net/latest/universe/types/${typeId}/`, {
            signal: controller.signal
        })
            .then(res => {
                if (!res.ok) throw new Error(`HTTP ${res.status}`);
                return res.json();
            })
            .then(json => {
                setData(json);
                setStatus("success");
            })
            .catch(err => {
                if (err.name === "AbortError") return;
                setError(err.message);
                setStatus("error");
            });

        return () => controller.abort();
    }, [typeId]);

    if (status === "loading") return <p>Lade Daten...</p>;
    if (status === "error") return <p style={{color:"red"}}>Fehler: {error}</p>;
    return (
        <div>
            <h2>{data?.name}</h2>
            <p>{data?.description}</p>
        </div>
    );
}

💡 Tipps

  • Verwende fetch direkt – kein spezieller HTTP-Service nötig.
  • Prüfe response.ok nach dem fetch, da fetch nur bei Netzwerkfehlern in den catch geht.
  • Der AbortController verhindert Speicherleaks und "State update on unmounted component"-Warnungen.
  • Nutze useEffect-Dependency-Array [typeId], damit bei Änderung der ID neu geladen wird.

✅ Akzeptanzkriterien

  • API-Daten werden asynchron geladen und angezeigt
  • Spinner wird während des Ladens angezeigt
  • Fehler werden abgefangen und als rote Meldung dargestellt
  • Der Request wird bei Unmount abgebrochen (AbortController)
  • useEffect hängt von typeId ab – bei Änderung wird neu geladen
