Aufgabe 04

Gekoppelte Inputs (Slider + Number)

🎯 Lernziel

Du kombinierst zwei verschiedene Input-Typen – einen <input type="range"> (Slider) und ein <input type="number"> – die denselben Wert steuern. Änderung am einen aktualisiert sofort den anderen. Du lernst bidirektionale Synchronisation mit useState umzusetzen.

🔧 Konzepte

<input type="range"> onChange useState Sync Math.round Props: min/max/step onValueChange callback

📋 Aufgabe

  1. 1. Erstelle DualInput.tsx mit einem Slider und einem Number-Input nebeneinander.
  2. 2. Beide Inputs binden an dieselbe useState-Variable. Änderung im Slider aktualisiert das Number-Feld und umgekehrt.
  3. 3. Verwende onChange für den Slider (flüssige Bewegung) und onChange für das Number-Feld (Präzisionseingabe).
  4. 4. Füge Props für min, max und step hinzu, damit die Komponente flexibel bleibt.
  5. 5. Gib den Wert per onValueChange-Callback nach oben.
  6. 6. Zeige den aktuellen Wert in Prozent-Format ("42,5 %") zwischen den beiden Inputs an.
💡 Hinweise – Code-Schnipsel

Die wichtigsten Bausteine:

// Slider (range) mit onChange für Echtzeit:
<input type="range"
       onChange={handleSliderChange}
       value={value}
       min={min} max={max} step={step} />

// Number-Input mit onChange für Präzision:
<input type="number"
       onChange={handleNumberChange}
       value={value}
       min={min} max={max} />

// Wert-Anzeige:
<span>{value.toFixed(1)} %</span>

// Bidirektionale Synchronisation:
const handleSliderChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const newVal = Number(e.target.value);
    setValue(newVal);
};

const handleNumberChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const raw = e.target.value;
    if (raw !== "") {
        const parsed = parseFloat(raw);
        if (!isNaN(parsed)) {
            setValue(Math.round(parsed * 10) / 10);
            onValueChange?.(Math.round(parsed * 10) / 10);
        }
    }
};

💡 Endlos-Schleifen vermeiden: Prüfe vor dem Callback, ob sich der Wert wirklich geändert hat.

🏁 Vollständige Lösung 
{/* DualInput.tsx */}

interface DualInputProps {
    min?: number;
    max?: number;
    step?: number;
    onValueChange?: (val: number) => void;
}

function DualInput({ min = 0, max = 100, step = 1, onValueChange }: DualInputProps) {
    const [value, setValue] = useState(50);

    const handleSliderChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        const newVal = Number(e.target.value);
        setValue(newVal);
        onValueChange?.(newVal);
    };

    const handleNumberChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        const raw = e.target.value;
        if (raw !== "") {
            const val = parseFloat(raw);
            if (!isNaN(val)) {
                const rounded = Math.round(val * 10) / 10;
                setValue(rounded);
                onValueChange?.(rounded);
            }
        }
    };

    return (
        <div className="d-flex align-items-center gap-3">
            <input type="range"
                   onChange={handleSliderChange}
                   value={value}
                   min={min} max={max} step={step} />

            <span className="fw-bold text-nowrap text-center"
                  style={{ minWidth: 80 }}>
                {value.toFixed(1)} %
            </span>

            <input type="number"
                   onChange={handleNumberChange}
                   value={value}
                   min={min} max={max} />
        </div>
    );
}

💡 Tipps

  • Achte auf Rundung: Der Slider liefert number (double), das Number-Feld auch. Verwende Math.round mit passender Genauigkeit.
  • Slider step sollte gröber sein (z.B. 1), das Number-Feld erlaubt beliebige Werte.
  • Vermeide unnötige Callbacks: Wenn sich der Wert nicht geändert hat, rufe onValueChange nicht auf.
  • Setze min/max auch als HTML-Attribute, damit der Browser den Slider korrekt skaliert.

✅ Akzeptanzkriterien

  • Slider und Number-Feld zeigen immer denselben Wert an
  • Slider-Bewegung aktualisiert das Number-Feld in Echtzeit
  • Number-Eingabe aktualisiert den Slider (ggf. gerundet auf Step)
  • Min, Max und Step sind per Props konfigurierbar
  • Der Wert wird als formatierter String (z.B. "42,0 %") angezeigt
  • Wert-Änderungen werden per Callback nach oben gemeldet
