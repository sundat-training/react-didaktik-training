Aufgabe 04b

Gekoppelte Inputs als Controlled Component

🎯 Lernziel

Du löst dieselbe Aufgabe wie in 04, aber als wiederverwendbare Controlled Component mit Props. Die Komponente bekommt value, onValueChange, min, max und step von außen – keine interne State-Verwaltung.

🔧 Konzepte

<input type="range"> <input type="number"> Props Callback Controlled Component Lifting State Up

📋 Aufgabe

  1. 1. Erstelle DualInput.tsx als Controlled Component mit Props value, onValueChange, min, max, step.
  2. 2. Der State lebt im Parent – die Komponente ruft nur onValueChange auf.
  3. 3. Beide Inputs (Slider + Number) zeigen value an und feuern onChange mit dem neuen Wert.
  4. 4. Verwende onChange für beide Inputs – React rendered bei jedem setState im Parent.
  5. 5. Zeige den Wert in Prozent-Format ("42,5 %") an.

📌 Der Unterschied zu 04

In 04 hast du useState innerhalb der Komponente verwendet. Hier liegt der State im Parent – die Komponente ist ein reines Presentational-Component. Das macht sie testbarer und wiederverwendbarer.

💡 Hinweise – Code-Schnipsel

Der Kern der Lösung:

// Props definieren:
interface DualInputProps {
    value: number;
    onValueChange: (val: number) => void;
    min?: number;
    max?: number;
    step?: number;
}

// Slider und Number – beide feuern denselben Callback:
<input type="range"
       value={value}
       onChange={(e) => onValueChange(Number(e.target.value))}
       min={min} max={max} step={step} />

<input type="number"
       value={value}
       onChange={(e) => {
           const raw = e.target.value;
           if (raw !== "") {
               const parsed = parseFloat(raw);
               if (!isNaN(parsed)) onValueChange(parsed);
           }
       }}
       min={min} max={max} />

// Im Parent:
const [value, setValue] = useState(50);
<DualInput value={value} onValueChange={setValue} />

💡 Der Parent steuert den Wert – die Komponente meldet nur Änderungen.

💡 Kein interner State, keine doppelte Wahrheit.

🏁 Vollständige Lösung 
{/* DualInput.tsx */}

interface DualInputProps {
    value: number;
    onValueChange: (val: number) => void;
    min?: number;
    max?: number;
    step?: number;
}

function DualInput({
    value,
    onValueChange,
    min = 0,
    max = 100,
    step = 0.5
}: DualInputProps) {
    const handleSliderChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        onValueChange(Number(e.target.value));
    };

    const handleNumberChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        const raw = e.target.value;
        if (raw !== "") {
            const parsed = parseFloat(raw);
            if (!isNaN(parsed)) {
                onValueChange(Math.round(parsed * 10) / 10);
            }
        }
    };

    return (
        <div className="d-flex align-items-center gap-3">
            <input type="range"
                   className="form-range flex-grow-1"
                   value={value}
                   onChange={handleSliderChange}
                   min={min} max={max} step={step} />

            <span className="fw-bold text-primary text-center"
                  style={{ minWidth: 80 }}>
                {value.toFixed(1)} %
            </span>

            <input type="number"
                   className="form-control"
                   style={{ width: 100 }}
                   value={value}
                   onChange={handleNumberChange}
                   min={min} max={max} step={step} />
        </div>
    );
}

// Verwendung im Parent:
function Parent() {
    const [value, setValue] = useState(50);
    return <DualInput value={value} onValueChange={setValue} />;
}

💡 Tipps

  • Da beide Inputs denselben value und denselben Callback nutzen, sind sie automatisch synchron.
  • onChange beim Slider feuert bei jeder Bewegung – React bündelt die Updates effizient.
  • Der Callback onValueChange wird vom Parent bereitgestellt – setValue von useState reicht oft direkt.
  • Verwende Math.round für konsistente Dezimalstellen.

✅ Akzeptanzkriterien

  • Slider und Number-Feld sind jederzeit synchron
  • Slider reagiert flüssig auf Ziehen
  • Number-Feld akzeptiert präzise Werteingabe
  • Die Anzeige zeigt den Wert in Prozent-Format
  • Value und Callback werden als Props übergeben – kein interner State
